# Task 1: Hospital Database Design

## Objective
To design and implement a relational hospital database system capable of managing patient records, doctors, appointments, prescriptions, and billing information.

## Database Tables
- Patient
- Doctor
- Department
- Appointment
- Prescription
- Billing

## Key Features
- Proper normalization
- Primary and foreign key constraints
- Referential integrity
- Realistic Indian patient data

## Files Included
- `Hospital_management_system.sql` – SQL script to create all tables
- `db_connect.py` – To connect with the DB using MySQL connector
- `insert_patients.py` – To insert fake patients data generated using faker
- `insert_doctors.py` – To insert fake doctors data generated using faker
- `insert_appointments.py` – To insert fake appointments data generated using faker
- `insert_departments.py` – To insert fake departments data generated using faker
- `insert_prescriptions.py` – To insert fake prescriptions data generated using faker
- `insert_billing.py` – To insert fake billing data generated using faker

## Outcome
A fully functional hospital management database ready for analytical and AI-based extensions.
