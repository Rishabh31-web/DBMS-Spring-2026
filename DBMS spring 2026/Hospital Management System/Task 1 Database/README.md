# Task 1: Hospital Database Design

## Objective
To design and implement a relational hospital database system capable of managing patient records, doctors, appointments, prescriptions, and billing information.

## Database Tables
- Patient
- Doctor
- Department
- Appointment
- Prescription
- Billing

## Key Features
- Proper normalization
- Primary and foreign key constraints
- Referential integrity
- Realistic Indian patient data

## Files Included
- `schema.sql` – SQL script to create all tables
- `sample_data/` – CSV files for each table

## Outcome
A fully functional hospital management database ready for analytical and AI-based extensions.
